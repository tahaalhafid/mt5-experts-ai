PURPOSE
Define explicit security-sensitive boundaries before any external delivery, integration, or broader productization.
This policy distinguishes secrets, credentials, runtime control surfaces, execution-contract material, internal-only operational surfaces, and externally non-deliverable artifacts.
This phase does not build a secret manager, encryption system, delivery integration, runtime auth engine, or broad code refactor.

SENSITIVITY_CLASSES
- SECRET_MATERIAL
  definition=raw secret values or secret-bearing containers whose exposure can compromise accounts, tokens, endpoints, or private channels
  examples=API_KEYS|WEBHOOK_SECRETS|PRIVATE_TOKENS|SECRET_CONFIG_PAYLOADS
- CREDENTIAL_MATERIAL
  definition=authentication or authorization material used to gain access to broker, bridge, delivery, or administrative systems
  examples=LOGIN_CREDENTIALS|ACCESS_TOKENS|SESSION_KEYS|BOT_TOKENS
- RUNTIME_CONTROL_SURFACE
  definition=artifacts that can shape or gate live runtime behavior, authority posture, or active truth interpretation if misused
  examples=AUTHORITATIVE_ACTIVE_PLAN|MUTATION_RULES|GOVERNOR_STATE|RUNTIME_GOVERNANCE_STATUS
- EXECUTION_CONTRACT_SURFACE
  definition=artifacts that define or constrain how execution-related behavior, control permissions, or bridge expectations are represented
  examples=AI_EXECUTION_CONTRACT|DIRECT_EXECUTION_PERMISSION_MAPS|CONTROL_CONTRACT_FILES
- INTERNAL_ONLY_OPERATIONAL_SURFACE
  definition=artifacts intended for internal operational use only because they contain raw state, raw diagnostics, internal journals, or sensitive context
  examples=RAW_DIAGNOSTICS|ROLLBACK_STATE|RAW_FEEDBACK_LOGS|RAW_MEMORY
- EXTERNALLY_NON_DELIVERABLE_SURFACE
  definition=any surface that must not be exposed in external delivery packages, advisory/reporting layers, or customer-facing outputs because exposure would leak secrets, credentials, control semantics, or unsafe raw context
  examples=SECRET_FILES|CONTROL_FILES|RAW_OPERATIONAL_STATE|UNSANITIZED_REPOSITORY_ARTIFACTS

FORMAL DEFINITIONS
secret_definition=any raw value or file content whose disclosure can grant unauthorized access, identify protected endpoints, expose trusted channels, or weaken operational security
credential_definition=any identity or authentication material enabling broker, transport, bridge, delivery, or administrative access
runtime_control_surface_definition=any surface capable of changing, gating, or misrepresenting authority, execution permissions, runtime posture, or active plan truth if exposed or misused
execution_contract_material_definition=any surface defining execution permissions, control semantics, bridge expectations, or execution-bound rules
non_deliverable_sensitive_artifact_definition=any artifact that contains raw secret, credential, control, or unsafe operational content and therefore may not enter external delivery paths in raw form

ISOLATION PRINCIPLES
- secret and credential material must remain outside AI intelligence, advisory, review, repository summary, and external delivery paths unless explicitly required for narrow internal runtime loading
- runtime control surfaces must remain separated from review artifacts, learning artifacts, advisory outputs, and external delivery materials
- execution-contract material must not be copied into reports, summaries, or customer-facing packages except as sanitized policy descriptions
- non-authoritative derived artifacts must never be mistaken for safe substitutes for raw control surfaces
- internal-only operational surfaces may be referenced by sanitized metadata or policy description only, not by raw payload disclosure

MINIMIZATION_RULES
- copy the minimum possible sensitive content for the minimum required internal purpose
- do not duplicate secret or credential values into repository history, AI prompts, review notes, advisory packets, experiment packages, or delivery outputs
- do not embed raw control-surface content into human-review artifacts when a policy reference or sanitized status summary is sufficient
- do not promote convenience copies of runtime control files into broader directories, archives, or delivery bundles
- when a surface is both useful and sensitive, preserve usefulness through sanitized abstraction rather than raw duplication

SANITIZATION_RULES
- operational summaries intended for external delivery must exclude raw secrets, raw credentials, control file contents, raw broker/account identifiers, and mutable runtime directives
- advisory, learning, review, and effectiveness artifacts must use sanitized references or classified placeholders when a sensitive source is relevant
- repository-visible summaries must remain non-authoritative and sanitized when sourced from sensitive operational material
- if sanitization is incomplete or uncertain, the artifact remains externally non-deliverable by default
- safe externalization requires deny-by-default treatment for raw sensitive payloads

AI_BOUNDARY_HYGIENE
- AI input policy must not grant raw access to secret-bearing files, credential-bearing files, runtime control surfaces, or execution-contract files unless a narrowly justified internal loader is explicitly required
- AI/review/advisory/reporting layers must not become indirect secret exfiltration paths through prompt stuffing, verbose summaries, copied raw logs, or disguised control-state disclosure
- learning, advisory, effectiveness, and review packages may use sanitized status references, bounded labels, or policy metadata only
- AI may reason about governance classes and sanitized statuses; AI may not inherit or emit raw secret or credential values
- repository and delivery artifacts must remain sanitized enough that external consumers cannot reconstruct control surfaces or runtime secrets

CONTROL_SURFACE_HYGIENE
- ai_current_plan.json remains authoritative active plan truth and must not be treated as external-deliverable material
- ai_execution_contract.json, ai_mutation_rules.json, runtime_governance_status.*, and related control files are control surfaces, not customer-facing knowledge artifacts
- backup, rollback, and raw state files remain internal-only and must not be used as external evidence bundles in raw form
- execution or governance control semantics may be described in policy language but must not be exported as raw actionable control payloads
