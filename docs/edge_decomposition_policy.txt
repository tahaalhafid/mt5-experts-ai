============================================================
WHAT DECOMPOSITION MEANS
============================================================
Decomposition means:
- transforming raw material into structured internal edge language
- separating narrative claims from operational mechanism
- separating thesis, trigger, invalidation, risk, and exit ideas into bounded fields
- preserving provisional regime/zone hints without pretending final licensing
- retaining lineage back to the registered source

Decomposition does NOT mean:
- approval
- codifiability success
- runtime value
- canon candidacy
- testing readiness
- activation readiness

============================================================
RAW-TO-NORMALIZED HANDLING RULES
============================================================
1) Runtime-imported family
   - normalize as family-level thesis + archetype + provisional zone/regime fit
   - preserve member-level dependency as a limitation
   - do not pretend family-level trigger detail is fully extracted yet

2) Runtime strategy implementation
   - normalize as a concrete strategy implementation placeholder
   - keep member title and runtime lineage
   - avoid inventing exact trigger/invalidation details not already captured

3) Legacy strategy blueprint / playbook / PDF / URL / note
   - normalize thesis, setup archetype, provisional trigger/invalidation/risk/exit fields where recoverable
   - preserve ambiguity when source precision is weak
   - allow PARTIAL / AMBIGUOUS / RESEARCH_ONLY outcomes

4) Indicator primitive
   - normalize primarily as confirmation/context/timing primitive
   - not as full strategy identity by default

5) Entry-pattern primitive
   - normalize primarily as timing/entry-shape primitive
   - not as full standalone edge identity by default

6) Filter primitive
   - normalize primarily as suppression/context/filter role
   - not as standalone edge identity

7) Risk-model primitive
   - normalize primarily as management / stop / exit / scaling primitive
   - not as thesis identity

8) Internal runtime finding / human review submission
   - normalize as attributable bounded material
   - require lineage to registered source and origin reference
   - do not allow narrative confidence alone to simulate structure

============================================================
ANTI-FABRICATION RULE
============================================================
At E1:
- preserve what is known
- mark unknowns clearly
- preserve ambiguity honestly
- do not fabricate trigger precision
- do not fabricate invalidation precision
- do not fabricate asset/timeframe specificity
- do not upgrade narrative into false structure

If a source cannot be decomposed cleanly, it must remain:
- DECOMPOSITION_PARTIAL
- DECOMPOSITION_AMBIGUOUS
- DECOMPOSITION_RESEARCH_ONLY
- DECOMPOSITION_BLOCKED_PENDING_BETTER_SOURCE

============================================================
LINEAGE PRESERVATION RULE
============================================================
Every normalized record must retain traceability back to:
- material registry record
- source registry record
- original origin_reference

No normalized record is allowed to become detached from lineage.
