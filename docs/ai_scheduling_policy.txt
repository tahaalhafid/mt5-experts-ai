SCHEDULING CATEGORY TAXONOMY

EVENT_IMMEDIATE
category_purpose=Run promptly for strong incident or candidate events when policy allows.
expected_usage=Critical incidents, execution-failure-linked forensic review, compact council advisory packets.
suppression_behavior=Same-scope duplicates suppressed within cooldown unless stronger evidence appears.
escalation_allowance=May escalate to TARGETED_RETRIEVAL or DEEP_INVESTIGATION only if the task family policy allows.
budget_sensitivity=High. Hard caps still apply.

EVENT_DEFERRED
category_purpose=Event-linked work that should not interrupt immediately and may defer to the next budget-safe window.
expected_usage=Non-critical drift checks, learning-governance evidence refresh, follow-up review tasks.
suppression_behavior=Defer or merge if a recent equivalent task already ran.
escalation_allowance=Escalate only when stronger repeated evidence appears.
budget_sensitivity=High.

HOURLY_WINDOW
category_purpose=Short cadence review window for bounded operational watch use.
expected_usage=Only for narrowly justified periodic review families.
suppression_behavior=If equivalent output already exists for the hour window, suppress rerun or emit compact update note.
escalation_allowance=No deep escalation by cadence alone.
budget_sensitivity=Medium.

DAILY_WINDOW
category_purpose=Primary cadence for drift, learning, and operating intelligence review.
expected_usage=Drift analysis, learning-governance support, bounded summary reviews.
suppression_behavior=Minor-noise triggers should defer into the scheduled daily window rather than forcing immediate reruns.
escalation_allowance=Targeted retrieval allowed if the daily review scope justifies it.
budget_sensitivity=Medium.

THREE_DAY_WINDOW
category_purpose=Broader institutional review window for aggregated evidence.
expected_usage=Three-day review inputs, broader learning support, bounded proposal preparation.
suppression_behavior=Do not rerun full three-day work for routine minor changes inside the same window.
escalation_allowance=Targeted retrieval allowed; deep escalation only for stronger evidence.
budget_sensitivity=Medium to high.

WEEKLY_WINDOW
category_purpose=Low-frequency governance review cadence.
expected_usage=Governance audit, broad control-surface review, institutional oversight summaries.
suppression_behavior=Routine signals already covered by the weekly scope should not retrigger the full audit.
escalation_allowance=Usually summary-first only; targeted retrieval only if a governance anomaly is material.
budget_sensitivity=Low to medium.

ON_DEMAND_ONLY
category_purpose=Explicit request driven work.
expected_usage=Operator copilot and tightly scoped support questions.
suppression_behavior=No background automatic reruns.
escalation_allowance=Targeted retrieval only when the request requires it and A1 allows it.
budget_sensitivity=Request-bound, not background-budget-driven.

REVIEW_TRIGGERED
category_purpose=Work that should run only when a formal review or proposal context exists.
expected_usage=Bounded proposal generation, review-board evidence packaging, institutional review bundles.
suppression_behavior=Suppress when the same review scope already has a current package and no material new evidence exists.
escalation_allowance=May use targeted retrieval when the review scope warrants it.
budget_sensitivity=High.

COOLDOWN AND DEDUPE RULES
- same_trigger_cooldown=If the same trigger family fires again inside AITriggerCooldownMinutes for the same scope, suppress rerun unless stronger evidence exists.
- same_family_cooldown=If the same task family already produced materially equivalent output for the same scope within its cooldown window, prefer suppression or compact update note.
- duplicate_incident_suppression=If an incident report already exists for the same decision_id, position_id, incident window, or candidate scope, do not rerun full analysis without stronger evidence.
- same_scope_suppression=If no material state change exists for the same scope, defer or suppress rerun.
- escalation_on_stronger_evidence_only=Only escalate severity or investigation depth when the new trigger materially strengthens evidence, broadens impact, or changes risk posture.
- rerun_deferral_no_material_change=If the state is materially unchanged, defer to the next cadence window rather than forcing another run.

EVENT VS CADENCE PRECEDENCE
- Event-immediate categories take precedence only for incident strength or candidate-linked contexts defined by policy.
- Cadence-led categories should absorb routine noise rather than rerunning on every minor event.
- Review-triggered work should not preempt higher-priority incident work under budget pressure.
- Operator on-demand requests remain eligible, but should still remain bounded in retrieval depth and output length.
- Council advisory is the only task intended for per-candidate immediacy, and even then only in compact summary form.

NON-GOALS
- no_runtime_scheduler
- no_runtime_queue
- no_counter_implementation
- no_delivery_integration
