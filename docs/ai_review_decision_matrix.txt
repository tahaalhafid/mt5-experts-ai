DECISION TYPES
- ACKNOWLEDGE
  allowed_roles=OPERATOR,ADMIN_REVIEWER,REVIEW_BOARD,COMMITTEE_REVIEWER
  meaning=Item has been seen and recorded without implying approval for runtime change.
  typical_next_state=INFORMATIONAL_ONLY or CLOSED_NO_ACTION

- REQUEST_MORE_EVIDENCE
  allowed_roles=ADMIN_REVIEWER,REVIEW_BOARD,COMMITTEE_REVIEWER
  meaning=Current evidence is insufficient for governed next-step acceptance.
  typical_next_state=DEFERRED or UNDER_REVIEW

- DEFER
  allowed_roles=ADMIN_REVIEWER,REVIEW_BOARD,COMMITTEE_REVIEWER
  meaning=Item remains potentially relevant but is postponed pending timing, data, or priority.
  typical_next_state=DEFERRED

- REJECT
  allowed_roles=ADMIN_REVIEWER,REVIEW_BOARD,COMMITTEE_REVIEWER
  meaning=No governed next step should proceed from the current artifact.
  typical_next_state=REJECTED

- ACCEPT_FOR_MONITORING
  allowed_roles=ADMIN_REVIEWER,REVIEW_BOARD,COMMITTEE_REVIEWER
  meaning=Item is accepted as watchlist or monitoring context only, without runtime mutation.
  typical_next_state=ACCEPTED_FOR_ACTION or CLOSED_NO_ACTION

- ACCEPT_FOR_EXPERIMENT
  allowed_roles=REVIEW_BOARD,COMMITTEE_REVIEWER
  meaning=Bounded experiment or bounded implementation-planning handoff is acceptable.
  typical_next_state=ACCEPTED_FOR_EXPERIMENT

- ESCALATE_TO_ADMIN
  allowed_roles=OPERATOR
  meaning=Item should move to ADMIN_REVIEWER because informational handling is insufficient.
  typical_next_state=ESCALATED

- ESCALATE_TO_REVIEW_BOARD
  allowed_roles=ADMIN_REVIEWER
  meaning=Item exceeds admin-only review scope and needs formal board review.
  typical_next_state=ESCALATED

- ESCALATE_TO_COMMITTEE
  allowed_roles=ADMIN_REVIEWER,REVIEW_BOARD
  meaning=Item has major governance significance or institutional contradiction.
  typical_next_state=ESCALATED

- CLOSE_NO_ACTION
  allowed_roles=ADMIN_REVIEWER,REVIEW_BOARD,COMMITTEE_REVIEWER
  meaning=Review cycle is complete with no further governed next step for the current item.
  typical_next_state=CLOSED_NO_ACTION

DECISION SAFETY RULES
- ACCEPT_FOR_MONITORING does not create live control authority.
- ACCEPT_FOR_EXPERIMENT does not auto-apply and does not equal runtime activation.
- No decision type in A7 authorizes direct BUY/SELL generation, direct execution override, direct plan mutation, or silent governor mutation.
- Escalation decisions change review venue only.

GOVERNED PATH SUMMARIES
- Proposal path = AI output -> PENDING_REVIEW -> ADMIN_REVIEWER triage -> REVIEW_BOARD required -> reject/defer/accept for experiment or governed action preparation -> handoff outside runtime auto-apply.
- Learning evidence path = evidence bundle -> review -> acknowledge/defer/reject/escalate/accept for experiment -> optional linkage to proposal -> no direct mutation.
- Incident path = incident/anomaly/drift/root-cause -> informational or pending review -> admin triage -> optional escalation -> close or link to evidence/proposal.
- Council advisory path = advisory packet -> visible/archive -> optional post-fact effectiveness review -> optional evidence linkage -> no live approval bureaucracy.

ESCALATION TRIGGERS
- repeated HIGH/CRITICAL incidents
- repeated drift deterioration
- anomaly clusters crossing multiple operational layers
- proposal/evidence artifacts with governance significance
- major governance contradiction
- recurring advisory hold patterns with measurable effect

CLOSURE CONDITIONS
- acknowledged informational artifact archived
- incident investigated and no action required
- proposal rejected and archived
- proposal accepted for experiment or governed action handoff and recorded
- evidence superseded by newer evidence
- drift signal expired without further significance
- escalated item received terminal decision from target reviewer role
