PURPOSE
Lock the MT5 dashboard architecture, page map, display-state vocabulary, source registry boundaries,
button model, widget contracts, cadence rules, and governance guardrails before any Phase 1 rendering work.

PHASE 0 HARD BOUNDARY
- no chart object rendering
- no live chart buttons
- no runtime action dispatch
- no request queue
- no mutation path
- no strategy/runtime/pilot/export posture change
- no authority expansion
- no operator intent creation

TOP-LEVEL POSTURE PRIORITY RULE
The primary posture banner must render, in order:
1. runtime posture
2. export posture
3. pilot posture
4. AI authority posture
5. factory posture

If multiple truths are simultaneously true, a higher-priority blocking or holding posture must remain visually dominant.
Positive bounded surfaces must never visually override runtime freeze, trade blocking, export blocking, or governance holds.

GLOBAL MIXED-PLANE WARNING RULE
If any page or widget mixes authoritative, derived, or placeholder/transitional states in one visible composition:
- show a page-level mixed-plane warning
- attach Authority Badge, Source Badge, and Freshness Badge
- mark summary-only surfaces as non-primary truth
- forbid any healthy/ready interpretation that hides blocking posture
